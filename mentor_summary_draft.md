Привіт! Хочу поділитись прогресом по проєкту Vehicle Cost Tracker (застосунок для відстеження витрат на доставку товарів — автопарк, водії, накладні, аналітика).

**Демо:** https://warehouse.mom/ — розгорнуто на власному Raspberry Pi (CI/CD пайплайн через GitHub Actions, деплой по Cloudflare Tunnel).

**Backend (Django REST API):** https://github.com/BohdanKyslii/vehicle_tracker_api
Готово: моделі (Car, Driver, Customer, Product, Waybill, RouteEvent та ін.), міграції, налаштування apps-структури. Адмінка, серіалізатори та views — наступний крок.

**Frontend (React PWA):** https://github.com/BohdanKyslii/vehicle_cost_tracker
Готово: лендінг, навігація, авторизаційна модалка, докер/nginx деплой на Pi.

І якщо в когось із студентів немає свого пет-проєкту — є другий напрямок у межах цього ж проєкту: облік складських витрат (оренда, комунальні, зарплати складу тощо). Це окремий Django-застосунок `warehouse` у тому самому бекенді, зі своїми моделями (не транспортними) — уже є робоча стара версія на HTML-шаблонах, яку треба перенести в окремий React SPA з новим DRF API, за тим самим підходом, що й transport-частина. Розгортатиметься на тому самому сайті (warehouse.mom), під своїм шляхом, зі спільною авторизацією. Бізнес-логіка й контракт API вже описані в документації проєкту. Пропоную одному зі студентів попрацювати над цим у парі зі мною — паралельно, над окремим від транспорту функціоналом.

Буду вдячний за будь-які рекомендації — особливо щодо структури Django-моделей, підходу до серіалізаторів/API-дизайну та архітектурних рішень на фронті.
